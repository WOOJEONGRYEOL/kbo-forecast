# KBO 단기 전력 예측 파이프라인 (kbo-forecast)

세이버메트릭스 기법으로 KBO 10개 구단의 **단기(최근 10경기) 전력**을 진단하고,
"곧 반등할 팀 / 곧 떨어질 팀"을 스크리닝하는 파이썬 파이프라인입니다.

> Gemini와의 기획 대화에서 설계한
> **"Rolling 피타고리안 기대승률 + 괴리율 스크리닝 + 투타 스킬 지표 결합"** 모델을
> 실제 데이터 소스에 연결해 완성한 것입니다.

## 실행 방법

```bash
# 1) 가상환경 만들기 (최초 1회)
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# 2) 실행
.venv/bin/python main.py                # 팀 단기 전력 예측 (최근 10경기)
.venv/bin/python main.py --window 15    # 윈도우 크기 변경
.venv/bin/python main.py --backtest     # 피타고리안 예측력 백테스트/캘리브레이션
.venv/bin/python players.py             # 투수/타자 개인 평가
```

### 백테스트 결과 (KBO 2026, point-in-time)

`--backtest`는 "최근 N경기 기대승률이 다음 N경기를 예측하는가"를 검증합니다.
- 10경기 윈도우: 과거 실제승률→미래 r≈0.00 vs 과거 기대승률→미래 r=+0.04 (피타고리안이 더 정확, 향상 +0.04)
- 예측력은 윈도우가 길수록 커짐 (20경기: r=0.25). **단기 예측은 노이즈가 크다**는 정직한 결론.

### 대시보드 인터랙션
- 선수 산점도: **5초마다 랜덤 선수 자동 순환** + 현재 선수를 흰 링으로 **하이라이트**. 클릭 시 그 선수 고정.
- 팀 대시보드: **분석 기준 슬라이더** — 최근 N경기(5~시즌최대, 임의 숫자)를 자유롭게 선택 + **시즌 전체** 토글.
  피타고리안·괴리율·모멘텀·운 산점도·시즌 흐름이 JS에서 **실시간 재계산**됩니다.
- **투수력 기준 토글**(전체 투수진 ↔ 선발 로테이션) — 켜면 구위+·모멘텀을 선발진 K-Stuff+로 재계산.
- 선발 로테이션 아스널 카드: 팀별 선발진의 K-Stuff+와 주무기 구종.
- 시즌 흐름 범례: 색상 대신 **팀 로고**(클릭 토글).

- `main.py` → 콘솔 진단표 + `data/report_날짜.csv` + **`data/dashboard.html`**
- `players.py` → 스크리닝 6종 + `data/pitchers_*.csv`, `batters_*.csv` + **`data/players.html`**

### 대시보드 인터랙션
- **지표 이름에 마우스를 올리면** 산정 공식이 툴팁으로 뜹니다 (팀·선수 양쪽).
- **선수 산점도는 아무 곳이나 클릭**하면 랜덤으로 한 명이 뽑혀 상세 카드가 표시됩니다.
  (점을 정확히 누르면 그 선수)
- **스크리닝 표는 기본 8명**만 보이고 '＋ 더 보기'로 전체 펼침/접기가 됩니다.
- 팀 진단표·모멘텀 막대는 **현재 시즌 순위 순(위=1위)**, 팀은 **로고**로 표시됩니다.
  로고는 네이버 엠블럼을 1회 받아 `data/logos/`에 캐시하고 HTML에 base64로 심습니다
  (네이버 CDN 핫링크 차단을 피하려고 — 파일을 옮겨도 로고가 유지됩니다).

⏱️ `players.py` 첫 실행은 시즌 전체 경기(400+)의 박스스코어를 받아오느라
몇 분 걸립니다. 이후에는 `data/box/` 캐시 덕에 새 경기만 받아옵니다.

## 모델 구조

```
[경기결과]  네이버 스포츠 API ──→ 팀별 경기로그 ──→ 최근 N경기 득실점
                                                      │
                                          피타고리안 기대승률 (지수 1.83)
                                                      │
                                          괴리율 = 기대승률 - 실제승률
                                                      │
[스킬지표]  kbostuff.app API ──→ 팀 K-Stuff+ (투수진 구위) ──┐
                            └──→ 팀 순수 wRC+ (타선 생산력) ──┤
                                                      │      │
                                        종합 모멘텀 = 0.25×z(괴리율)
                                                   + 0.50×z(구위+)
                                                   + 0.25×z(타선+)
```

> **가중치 근거 (2026-07 갱신)**: 예전엔 괴리율에 0.5를 줬으나,
> 2021~2025 5시즌 point-in-time 백테스트에서 과거 성적을 통제했을 때
> 구위 항의 기여(편상관 +0.08~0.13)가 괴리율(+0.03~0.06)의 2~3배로
> 나와 비중을 뒤집었습니다. `python main.py --skill-backtest`로 재현됩니다.
> (타선 항은 경기 단위 스킬 로그가 없어 검증 불가 → 0.25 유지)

**핵심 아이디어**: 최근 성적에서 '운'을 걷어내고 '실력의 방향성'만 남기는 것.

- **괴리율 (+)** — 득실점 마진 대비 승수가 적음 = 운이 없었음 → 반등 후보 📈
- **괴리율 (−)** — 득실점 마진 대비 승수가 많음 = 운이 좋았음 → 하락 경계 📉
- 여기에 시즌 누적 **K-Stuff+**(결과가 아닌 공 자체의 위력)와
  **순수 wRC+**(파크팩터·비거리 보정 타격 생산력)를 섞어
  "운의 되돌림이 어느 방향으로 얼마나 강할지"를 보정합니다.

## 선수 개인 평가 모델 (players.py)

Gemini 기획 대화의 [추천 1: 피칭 모델링]과 [추천 2: 타자 스킬 분류]의 구현입니다.

**투수 — 구위(원인) × 성적(결과) 사분면**

|  | 성적 좋음 | 성적 나쁨 |
|---|---|---|
| **구위 좋음 (K-Stuff+ ≥105)** | ✅ 진짜 에이스 | 📈 억울한 투수 (반등 후보) |
| **구위 나쁨 (K-Stuff+ ≤97)** | ⚠️ 시한폭탄 (하락 경계) | 😞 부진 |

- 성적(ERA/FIP)은 네이버 **경기별 박스스코어를 시즌 합산**해 직접 계산합니다.
  (KBO 공식 페이지는 크롤러 POST 차단, 스탯티즈는 로그인제라서)
- 박스스코어의 선수코드(pcode)가 kbostuff와 같은 체계라 정확한 조인이 가능합니다.
- ERA−FIP 격차로 '수비/운 피해자' 순위도 뽑습니다.

**타자 — 운 진단 + 유형 분류**

- **BABIP 운 진단**: 리그 평균 BABIP에서 ±0.040 이상 벗어나면 💎저평가/🫧거품 후보.
  (원래 xwOBA 잔차를 쓰려 했으나 kbostuff의 xwOBA는 wOBA와 상관 0.2 수준이라 부적합 판정)
- **파워 유형**: Power+(장타) × HR+(홈런 파워) 교차 → 갭 파워 / 홈런 스페셜리스트 / 컴플리트 파워
- **구장 피해자**: 순수 wRC+ − 이벤트 wRC+ 격차 (잠실·대구 등 큰 구장에 갇힌 타자)

**주의사항**

- 시범경기 오염 방지: 네이버 일정 API는 3월 시범경기를 함께 주므로,
  경기별 `gameFlag`(시범=1)로 걸러냅니다 (`data/gameflags_*.json`에 캐시).
- 네이버 박스스코어의 부분 이닝은 유니코드 분수(⅓, ⅔)로 표기됩니다 — 파싱 주의.
- BABIP 스크리닝은 후보 목록일 뿐입니다. 발 빠른 타자는 실력으로 높은 BABIP를 유지합니다.

## 데이터 소스

| 소스 | 용도 | 방식 |
|---|---|---|
| 네이버 스포츠 API | 경기별 득점/실점/승패 | 공개 JSON API, 인증 불필요 |
| kbostuff.app (KBO Talent) | K-Stuff+, Batter Metrics+, 순수 wRC+ | Supabase REST API (공개 anon 키) |

처음 기획에서 후보였던 **스탯티즈는 2026년 현재 일정 페이지가 로그인제로 바뀌었고**,
KBO 공식 홈페이지의 Schedule 엔드포인트도 막혀 있어 위 조합으로 확정했습니다.

kbostuff.app의 API는 브라우저 개발자도구 [Network] 탭에서 사이트가 실제로 호출하는
Supabase 엔드포인트를 관찰해 재현한 것입니다. 사용 테이블:

- `pitching_metrics_v2_leaderboard` — 투수별 K-Stuff+ v2 / K-Location+ v3
  (로케이션은 `pitch_type_details` JSON의 edge/heart/waste에서 추출)
- `pitching_metrics_v2_game_log` — **경기별** K-Stuff+ (2021~, 시점별 스냅샷).
  모멘텀 가중치 백테스트와 팀 구위의 트레이드-강건 집계에 씁니다.
- `batter_metrics_plus` — 타자 종합+ 지표 (팀 코드 포함)
- `regular_batter_advanced` — 순수 wRC+ (`wrc_plus_pure`), WAR (`war_pure` 등)
- `players` — 선수코드 → 소속팀 매핑 (투수용, 전체 페이지네이션 수집)
- `fcb_season_leaderboard` — 타자 FCB 승리기여 (설명형 지표)

## 알아둘 한계 (정직 코너)

1. **10경기 윈도우는 표본이 작습니다.** 괴리율 ±0.05 임계값은 관례적인 값이고,
   실제 예측력은 과거 시즌으로 백테스트해서 캘리브레이션하는 것이 정석입니다.
2. **스킬 지표는 시즌 누적치입니다 — 그리고 그게 옳았습니다.** K-Stuff+·wRC+는
   '최근 10경기'가 아니라 시즌 전체 값이라 최근 폼 변화가 늦게 반영됩니다.
   그런데 이걸 최근 N경기로 바꿔 봤더니(`stuff_mode="recent"`) 오히려
   예측력이 떨어졌습니다(모든 윈도우에서 r −0.04). K-Stuff+는 안정적 스킬
   지표라 표본이 많을수록 정확하기 때문입니다. **누적이 약점이 아니라 강점이었습니다.**
   (단, 팀 집계는 이제 게임로그 기반이라 시점별 재계산·백테스트가 가능합니다)
3. **트레이드 왜곡 — 팀 집계는 해결됨.** 팀 단위 구위는 이제 경기별
   게임로그에 네이버 박스스코어의 경기별 팀을 조인해 씁니다(`team_stuff_by_game`).
   `players.current_team_code`(현재 팀) 의존이 사라져, 이적 선수의 과거 기록이
   옛 팀에 정확히 남습니다. (개인 평가의 성적 지표는 여전히 박스스코어 기반)
4. **모멘텀 가중치는 이제 데이터로 정했습니다.** 0.5/0.25/0.25 → 0.25/0.50/0.25.
   `config.MOMENTUM_W_*`에서 조절하며, 근거는 config.py 주석과 위 모델 구조 참고.
   ⚠️ 그래도 절대 예측력은 약합니다(미래 R² 0.02~0.05). 단기 야구의 본질적 한계입니다.
5. **K-Stuff+ 스케일 드리프트 방어.** kbostuff가 상류에서 스케일을 재정규화하면
   (2026시즌만 4/3·6/19 두 번, 7/16 이후 리그 평균이 100→108로 급등) 절대값이
   흔들립니다. 그래서 게임로그는 '그날 리그 평균 대비'로 재센터링해서 씁니다
   (`team_stuff_by_game(recenter=True)`).

## 로케이션(K-Location+) 결합 — 투수 2축 평가

kbostuff가 2축의 한 축으로 내세우는 로케이션은 사실 `pitch_type_details`
JSON 안에 이미 받고 있었지만(구사율·구종명만 꺼내 쓰고 나머지는 버림),
이제 `summarize_location()`으로 살려 씁니다. 추가 API 호출은 없습니다.

- **Gap 축** `stuff_loc_gap = K-Stuff+ − K-Location+` — '억울한 투수'의
  원인을 가릅니다. Gap이 크면 제구 교정 여지(🎯 고칠 수 있음), 작으면 순수 불운.
- **🎯 제구만 잡으면 반등** 스크리닝: 구위 최상급 + 로케이션 부진 + ERA 나쁨.
- **⚠️ 한가운데 실투 경계** 스크리닝: `heart_pct` 높은 투수 = 아직 안 터진
  피장타 예비군 (선행지표).
- 아스널 카드가 구종별로 구위·**로케이션·한가운데 실투율**까지 표시합니다.

## 시즌 최종 순위 예측 (`--standings-sim`)

단기 예측은 약하지만(다음 10경기 R²≈0.01), **최종 순위는 잘 맞습니다.**
시즌의 60%+가 이미 확정됐고 남은 40%만 불확실하기 때문입니다.

- **잔여 매치업 복원**: KBO는 팀당 각 상대와 정확히 16경기(9팀×16=144)라,
  지금까지 팀쌍별 대전 수만 세면 남은 상대·횟수가 규칙으로 완전히 복원됩니다
  (홈8·원정8까지). **잔여 일정(날짜)이 아직 안 나와도 순위 확률은 안 바뀝니다.**
- **방법**: 팀 강도(피타고리안) → log5 + 홈 어드밴티지 → 남은 경기 몬테카를로
  20,000회 → 1위 확률 / 가을야구(top5) 확률 / 순위 90% 구간.
- **백테스트로 검증** (2021~2025, 5시즌):

  | 진행률 | 순위 상관(스피어만) | 가을야구 적중 |
  |---|---|---|
  | 50% | 0.79 | 3.8/5 |
  | 61% | 0.80~0.82 | 3.8~4.0/5 |
  | 75% | 0.90~0.92 | 4.4/5 |

  강도는 피타고리안이 현재 승률보다 낫거나 비슷합니다(시즌 초반 1위 적중
  피타 100% vs 현재 80% — 운으로 앞선 팀을 득실점이 걸러냄).

## 검증하고 기각한 것들 (음의 결과도 결과다)

가설이 틀렸다는 것도 재현 가능한 결론이라 코드로 남겨둡니다.

- **예측 지평 확대** (`--skill-backtest`의 지평 표). "다음 5경기"는 사실상
  못 맞춥니다(R²≈0.01). 지평을 20~30경기로 늘리면 R²가 3~5배가 됩니다.
  그래서 모델 판정은 '다음 경기'가 아니라 **'향후 20~30경기 방향성'**으로
  프레이밍합니다. 40경기까지 늘리면 정확하지만 '원래 좋은 팀이냐'로 수렴해
  실행 가치가 떨어집니다.
- **선발 매치업 예측 — 기각** (`--matchup-backtest`). "다음 경기 예고 선발의
  구위차로 승패를 예측한다"를 검증했더니, 선발 구위차(AUC 0.471)가 팀 최근
  폼(0.547)보다 **못했습니다.** 구위 우위 선발을 낸 홈팀이 오히려 덜 이겼습니다
  (0.497 vs 0.536). 선발은 경기의 일부일 뿐이고 팀 폼이 이미 투타·불펜을
  종합 반영하기 때문입니다. + KBO 선발 예고는 **당일분만** 확정돼(전날 밤,
  네이버 `/preview`) 하루 1회 배치와도 궁합이 나쁩니다. → 당일 매치업 카드
  미구현.
